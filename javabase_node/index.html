    封装性

    隐藏对象内部的复杂性，之对外暴露简单的接口。便于外界调用，从而提高系统的可扩展性可维护性

    • 低耦合：仅对外暴露少量的方法用于使用

    继承性

    一旦子类继承父类后，父类中的所有属性和方法都将被属性所获得，私有的也不例外。但是由于封装性的影响，子类只能通过父类的公有方法调用其私有变量或方法。


    方法的重写

    子类继承父类以后，可以对父类中同名同参的方法进行覆盖操作

    重写即覆盖，覆盖即说明子类的“范围”要比父类大

    • 子类中的重写方法的权限修饰符需要大于等于父类中被重写方法的权限修饰符
    • 子类不能对父类的私有方法进行重写
    • 父类被重写的方法的返回值类型为void，则子类重写的方法的返回值类型只能是void
    • 父类被重写的方法的返回值类型为A类型，则子类重写的方法的返回值类型只能是A类型或A类型的子类
    • 父类被重写的方法的返回值类型为基本数据类型，则子类重写的方法的返回值类型只能是相同的基本数据类型
    • 子类重写方法抛出的异常不大于父类被重写方法抛出的异常类型；如果父类被重写方法抛出异常，子类重写方法要么不抛异常，要么抛出该异常或该异常的子类

    子类和父类中同名同参的方法要么都声明为非static（考虑重写），要么都声明为static


    super关键字

    super调用构造器

    条件：

    • super(形参列表)必须在子类构造器的首行
    • 在子类的构造器中，this(形参列表)和super(形参列表)只能二选一，不能同时出现
    • 在子类的构造器中，如果没有显式的声明this(形参列表)或super(形参列表)，则默认调用的是super()

    D8M9（面向对象）

    子类对象实例化的全过程

    1. 从结果上来看（继承性）：
      • 子类继承父类以后，就获取了父类中声明的属性和方法
      • 创建子类对象，在堆空间中，就会加载所有父类中声明的属性
    2. 从过程上来看：
      • 当通过子类构造器来创建子类对象时，会直接或间接的调用其父类的构造器，进而调用父类的父类的构造器，直到调用了java.lang.Object类中空参的构造器为止。正因为加载过所有父类的结构，所以才可以看到内存中有父类的结构，子类对象才可以考虑进行调用。 总之就是实例化子类时把所有父类的属性和方法进行初始化供其子类调用
      • 虽然创建子类对象时调用了父类的构造器，但是自始至终就创建过一个对象，即new的对象

    多态性

    1. 理解多态性：可以理解为一个事物的多种形态
    2. 何为多态性：对象的多态性 --> 父类的引用指向子类的对象
    3. 多态性的使用，虚拟方法调用：在编译期，只能调用父类中声明的方法；在运行期，执行的是子类重写父类的方法。 总结：编译看左边，运行看右边
    4. 多态性的使用前提：
      • 有类的继承关系
      • 有方法的重写
    5. 对象的多态性，只适用于方法，不适用于属性；调用属性仍是父类的

    虚拟方法调用（多态情况下）：子类中定义了与父类同名同参数的方法， 在多态情况下，将此时父类的方法称为虚拟方法 ，父类根据赋给它的不同子类对象，动态调用属于子类的该方法，这样的方法调用在编译器是无法确定的

    编译时和运行时类型（多态情况下）： 编译时对象类型看左边，运行时类型看右边 ；方法的调用是在运行是确定的

    早绑定和晚绑定：对于重载而言，在方法调用之前，编译器就已经确定了要调用的方法，这称为“早绑定”或者“静态绑定”；对于多态而言，只有等到方法调用的那一刻，解释运行器才会确定所要调用的具体方法，这称为“晚绑定”或“动态绑定”；如果不是“晚绑定”，就不是多态

    单例设计模式

    设计模式是在大量的实践中总结和理论化之后优选的代码结构、编程风格、以及解决问题的思考方式。免去我们自己再思考和摸索

    对于某个类，只能存在一个对象实例，所以要将该类的构造器的访问权限设置为private，这样，就不能在外部创建这个类的实例。在内部需提供一个静态方法来return该类的实例供外部调用，这个实例也需要设置为静态，这样实例才能被静态方法调用

    //好处：线程安全

    //坏处：对象加载时间过长

    class SingletonTest01 {

      private static SingletonTest01 st = new SingletonTest01();

      private SingletonTest01(){};

      public SingletonTest01 getST(){

        return st;

      }

    }

    //懒汉式

    //好处：延迟对象的创建

    class SingletonTest02 {

      private static SingletonTest02 st = null;

      private SingletonTest02(){};

      public SingletonTest02 getST(){

        if(st == null) st = new SingletonTest02();

        return st;

      }

    }

    理解main方法的语法

    main方法的使用说明

    1. main方法作为程序的入口
    2. main方法也是一个普通的静态方法，也可以被其他的方法调用
    3. main方法可以作为与控制台交互的方式

    代码块

    1. 代码块的作用：用来初始化类、对象
    2. 代码块如果有修饰，只能用static修饰

    静态代码块

    • 内部可以有输出语句
    • 随着类的加载而执行，而且只加载一次
    • 如果一个类中定义了多个静态代码块，则按从上到下的先后顺序执行
    • 属性：初始化类的信息

    非静态代码块

    • 内部可以有输出语句
    • 随着对象的创建而执行
    • 每创建一个对象，就执行一次非静态代码块
    • 作用：在创建对象时，对对象的属性进行初始化

    属性赋值的先后顺序

    1. 默认初始化
    2. 显式初始化/在代码块中赋值
    3. 构造器中初始化
    4. 有对象后，通过调用属性或者方法赋值

    final关键字

    1. final可以用来修饰的结构：类、方法、变量
    2. final用来修饰一个类：此类不能被其他类基础
    3. final用来修饰方法：表示此方法不能被重写
    4. final用来修饰变量，此时的变量就称为是一个常量
      • final修饰属性，可以考虑赋值的位置有：显式初始化、代码块中初始化、构造器中初始化
      • final修饰局部变量
    5. static final用来修饰属性：全局常量

    abstract关键字

    1. abstract：抽象的
    2. abstract可以用来修饰的结构：类、方法
    3. abstract修饰类：抽象类
      • 此类不能实例化
      • 抽象类中一定有构造器，便于子类实例化调用
    4. abstract修饰方法：抽象方法
      • 抽象方法只有方法的声明，没有方法体
      • 子类重写父类中所有抽象方法后才能实例化

    2. 如何定义接口，定义接口中的成员
      1. JDK7及以前：只能定义全局常量和抽象方法

    JDK8中接口的小知识点

    1. 如果子类（或实现类）继承的父类和实现的接口中声明了同名同参数的默认方法，那么子类在没有重写方法的前提下，默认调用的是父类中的同名同参的方法 ---> 类优先原则
    2. 如果实现类实现了多个接口，而这多个接口中定义了同名同参数的默认方法，那么在实现类没有重写此方法的情况下，报错 ---> 接口冲突，此时需要重写该方法
    3. 在知识点2中，实现类中重写方法后如何调用不同接口中被重写的同名同参的方法？ ---> 接口.super.method();

    内部类

    类的内部成员：属性，方法，构造器，代码块，内部类；别学完了都不知道类的内部成员

    1. Java允许将一个类A声明在另一个类B中，则类A就是内部类，类B就是外部类
    2. 内部类的分类：成员内部类（静态、非静态），局部内部类（方法内、构造器内、代码块内）
    3. 成员内部类：
      • 一方面，作为外部类的信息
        • 调用外部类的结构
        • 可以被static修饰
      • 一方面，作为一个类
        • 类内可以定义属性、方法、构造器
        • 可以被final修饰
        • 可以被abstract修饰
      • 如何实例化成员内部类
        • 静态的
          • 外部类.内部类 变量名 = new 外部类.内部类();//静态的
        • 非静态的
          • 创建外部类对象
          • 外部类.内部类 变量名 = 外部类对象实例.new 内部类();
      • 如何在成员内部类中区分调用外部类的结构
        • 属性：外部类.this.属性
        • 方法：外部类.this.方法()
    4. 在局部内部类的方法中，如果调用局部外部类所声明的方法中的局部变量，要求此局部变量声明为final
      • JDK7及之前的版本，要求此局部变量显式的声明为final
      • JDK8及之后的版本，可以省略final的声明
